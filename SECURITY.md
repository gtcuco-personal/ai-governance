# Security Policy

## Reporting Vulnerabilities

- **Preferred:** GitHub Private Vulnerability Reporting
- **Alternative:** Direct contact via repository owner
- **Do NOT** open a public GitHub issue

## Sensitive Information Policy

- **Never commit** credentials, API keys, or personal data without proper handling
- Content is for internal use only — review before sharing

## Sensitive Files

- `.env` / any file with API keys — in `.gitignore`, never committed
